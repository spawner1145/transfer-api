Transfer unlimited.surf API

OpenAI / Anthropic compatible adapter for https://unlimited.surf. Run it locally with one Node command, or deploy anywhere that runs Node.js \u2014 no Cloudflare / wrangler required.

1. Pick a deployment target

This page is hosted on GitHub Pages and is static only: it cannot proxy API calls itself. Run the proxy in one of these places, then point this page (or your real client) at it.

Run locally (recommended for personal use)

git clone https://github.com/<you>/transfer-api.git
cd transfer-api
# No npm install needed \u2014 the server has zero runtime dependencies.
UNLIMITED_SURF_API_KEY=<your unlimited.surf key> \
WORKER_API_KEY=<optional client-facing key> \
node server.js

Default URL: http://localhost:8787

Run with Docker

docker build -t transfer-api .
docker run -p 8787:8787 \
  -e UNLIMITED_SURF_API_KEY=<your unlimited.surf key> \
  -e WORKER_API_KEY=<optional client-facing key> \
  transfer-api

Other hosts

  • Render / Railway / Fly.io / Koyeb: deploy from this repo, set node server.js as the start command, expose port 8787 (or read $PORT).
  • Vercel / Netlify Functions: wrap handleRequest from src/worker.js in their fetch-style adapter.
  • Cloudflare Workers: still supported via npx wrangler deploy, but no longer required.
